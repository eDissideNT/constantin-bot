# Constantin Bot — Plan

## Context

Telegram-бот-конструктор, который создаёт инфраструктуру для других ботов.

## Технические решения
- TypeScript, библиотека Grammy
- NocoDB, PostgreSQL
- Логика самих новых ботов написана на Python: `bots/notification_bot.py`
- Запуск Python-ботов: PM2 (через Node.js API, пакет `pm2` + `@types/pm2`)
- Реестр ботов: PostgreSQL (без интерфейса в NocoDB)
- Инфраструктура: docker-compose (PostgreSQL, NocoDB, constructor bot)

## Команды бота
- `/add` — создать нового бота
- `/list` — список всех ботов со статусами (`running` / `stopped`)
- `/stop <bot>` — остановить бота
- `/run <bot>` — запустить остановленного бота
- `/delete <bot>` — удалить бота (остановить процесс, удалить таблицу NocoDB, убрать из реестра)

### Сценарий добавления нового бота (команда /add)
1. Админ пишет в бот-конструктор команду /add
2. Бот пошагово запрашивает: токен от BotFather, имя бота
3. Бот создаёт таблицу в NocoDB для нового бота
4. Бот запускает программу (Python-бот из папки bots) нового бота
5. Возвращает админу ссылку на созданную таблицу и адрес нового бота

## Реестр ботов (PostgreSQL)

Таблица `bots`:

| Поле               | Тип | Описание             |
|--------------------|---|----------------------|
| id                 | serial PK |                      |
| name               | varchar | Имя бота             |
| token              | varchar | Telegram-токен       |
| messages_table_url | varchar | URL таблицы в NocoDB |
| status             | varchar | `running` / `stopped` |
| created_at         | timestamp | Дата создания        |

## Структура таблицы NocoDB

Таблица — это сценарий воронки: последовательность сообщений, которые бот отправляет пользователю.
Эталон: `data/LeoMarquez_bot.xlsx`, лист «Основной».

| Колонка | Название | Тип | Описание |
|---|---|---|---|
| A | Тип сообщения | string | `Кружок`, `Текст`, `Медиа` |
| B | Тип времени | string | Единица задержки (`сек`) |
| C | Время | number | Задержка в секундах. `-1` = ждать действия пользователя |
| D | Текст | string | Текст сообщения (или `-`) |
| E | Медиа | string | Telegram file_id (или `-`) |
| F | Наличие кнопки | string | `Да` / `Нет` |
| G | Текст кнопки 1 | string | Текст первой кнопки (или `-`) |
| H | Содержание кнопки 1 | string | Номер строки перехода или URL (или `-`) |
| I | Текст кнопки 2 | string | |
| J | Содержание кнопки 2 | string | |
| K | Текст кнопки 3 | string | |
| L | Содержание кнопки 3 | string | URL или номер строки |
| M | Ожидать сообщения юзера | string | Ждать ответ пользователя (или `-`) |
| N | Реакция на кнопку | string | Действие при нажатии (или `-`) |
| O | Ссылка кнопки | string | URL кнопки |

Кнопки: содержание — число = переход на строку сценария, URL = inline-ссылка.
